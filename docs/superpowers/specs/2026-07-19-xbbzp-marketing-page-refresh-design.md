# 响帮帮官网(xbbzp.html)内容重构 — 设计文档

- 日期：2026-07-19
- 状态：已确认设计，待写实现计划

## 背景

`web/public/xbbzp.html` 是响帮帮对外营销首页，单文件 hash 路由（不经 Vite 处理，`public/` 下的文件原样拷进 `web/dist`），已在 `backend/app.py` 的 `_WEB_ROOT_FILES` 白名单里挂在根路径。登录页 `LoginView.vue` 的"返回官网"链接已经指向它。

现状（`b4be35f` 起未变）：首页文案定位是"劳务派遣·工地仓储物流"一类描述，`#/baojingyun` 二级页详细介绍响帮帮保经云（参保管理云平台）三端登录入口。

真实的公司官网 `https://www.xbbzp.com` 是另一个 Vue SPA（`<title>响帮帮 - 一站式可信灵活用工交易平台</title>`），用 Playwright 渲染后拿到真实定位与文案，与 `xbbzp.html` 现有内容不一致：

- 品牌定位：**"一站式可信灵活用工交易平台"／"零工界的嘀嘀"**
- 四件核心事：人岗智能速配、透明交付、合规结算、证据留存
- 产品端：企联端／灵工端／运管端／开放平台
- 覆盖行业：制造业／建筑业／农牧业／家政业／商贸服务业
- 三类目标客群：用工方／劳动者／行业伙伴
- 三个客户案例：制造业（人力成本降低30%、结算效率提升5倍）、物业管理（覆盖20+城市、对账时间缩短80%）、农牧采摘（结算纠纷降至0、管理效率提升3倍）
- 真实联系方式：176 813 17237 / xiangbangbang@126.com / www.xbbzp.com
- 备案信息：豫ICP备19004802号-1、豫公网安备 41019702002797号

本次是把 `xbbzp.html` 的**内容**换成上述真实定位与文案，**视觉风格保持现有的 navy+amber 配色 / mono+CJK 字体 / ticket 卡片质感 / 深色模式适配不变**——不套用真实官网的蓝色系。响帮帮保经云在整个站点里维持"旗下产品/二级页面"的从属关系（本来就是这个结构，不用改）。

## 目标

1. 首页内容与真实官网 `xbbzp.com` 的当前定位、四件事、客群、案例、联系方式对齐。
2. 导航结构从现有的 3-view（首页/保经云/登录锚点）扩展为贴近真实官网"多页感"的 5-view，响帮帮保经云仍以"旗舰产品"身份出现在其中一条业务线/能力下，不喧宾夺主。
3. 保持现有 navy+amber 视觉系统、单文件 hash 路由的工程方式不变，不引入新的构建步骤。

## 范围边界

- 不改 `backend/app.py`（文件名仍是 `xbbzp.html`，白名单无需变）。
- 不改 `LoginView.vue` 或任何 Vue 路由/组件——这次改动完全在 `web/public/xbbzp.html` 一个静态文件内。
- 不改 `#/baojingyun` 二级产品页的内容主体（已经准确详细），只调整其顶部导航链接以指向新的多页结构。
- 统计数字（覆盖行业/平台劳动者/累计结算规模）沿用现有"示例数字 + 正式上线前请替换"的注释方式，不编造真实经营数据。
- 不涉及实际生产部署——本次设计只覆盖内容改造；build/测试/合并/push 触发 Render 部署这几步在实现完成后单独确认执行。

## 页面结构（5 个 hash view）

### `#/` 首页

- Hero：eyebrow"一站式可信灵活用工交易平台"，主标题"响帮帮 / 零工界的嘀嘀"，lede 引用四件事概括，CTA 指向 `#/capabilities` 和 `#/baojingyun#access`（直接登录）。
- 沿用现有 ticket 卡片视觉元素（岗位/参保凭证质感）作为 hero 配图。
- 业务线三卡（沿用现有 `.triad` 组件）：灵活用工能力（→ `#/capabilities`）／**响帮帮保经云·旗舰产品**（→ `#/baojingyun`，视觉上保持 flagship 样式）／资金结算能力。
- 统计条：覆盖行业 / 平台劳动者 / 累计结算规模，沿用现有示例数字 + 替换提示注释。
- 底部 CTA 区引导到 `#/about` 联系方式。

### `#/capabilities` 产品能力

- 四件事详解（人岗智能速配、透明交付、合规结算、证据留存），沿用现有 `.feat-grid` 卡片组件改写文案。
- 四端说明：企联端／灵工端／运管端／开放平台。
- "合规结算"能力下重点带出响帮帮保经云作为旗舰产品的 spotlight 区块（沿用现有 `.spotlight` 组件），链接到 `#/baojingyun`。

### `#/solutions` 解决方案

- 行业分类：制造业／建筑业／农牧业／家政业／商贸服务业（卡片列表，复用 `.feat` 或新增同风格小卡）。
- 客户成功案例三个（制造业/物业管理/农牧采摘），每个案例带两个数据点（复用现有 `.stat` 风格的小型双数字展示）。

### `#/about` 关于我们

- "为谁服务"三类客群卡片：用工方／劳动者／行业伙伴，各自的价值主张。
- 联系方式与备案信息区块：电话、邮箱、官网域名、豫ICP/公安备案号。

### `#/baojingyun` 响帮帮保经云（基本不变）

- 内容主体保留现状（hero、7 个 feat 卡、4 步 flow、三端登录入口 `#access`）。
- 仅将顶部导航 `.nav-links` 从"首页/保经云/登录入口"改为与新结构一致："首页/产品能力/解决方案/关于我们/登录入口"，并把"返回响帮帮首页"链接改为 `#/`。

## 导航与 Footer

- Footer 换成真实联系方式（176 813 17237、xiangbangbang@126.com、www.xbbzp.com）+ 备案号（豫ICP备19004802号-1、豫公网安备 41019702002797号），沿用现有 footer 布局，不新增组件。

## 视觉规范（不变）

- 配色变量沿用现有 `:root` 定义（navy/amber/paper/steel 等），含 `:root[data-theme="dark"]` 与 `prefers-color-scheme: dark` 两套覆盖。
- 字体沿用 `--cjk` / `--mono`。
- 复用现有 CSS 组件类名（`.hero`、`.triad`、`.feat-grid`、`.spotlight`、`.badges`、`.stats`），新增 view 只新增少量必要的类，不引入新的设计语言。

## 测试与验证

- 无自动化测试覆盖（纯静态营销页），验证方式：`cd web && npm run build` 无报错，本地打开 `web/dist/xbbzp.html`（或开发服务器）人工检查 5 个 view 的导航跳转、响应式断点（900px/640px）、亮暗色模式显示是否正常。
- 确认登录页 `LoginView.vue` 里"返回官网" `/xbbzp.html` 链接行为不受影响（跳到 `#/` 首页）。

## 发布

设计范围只到内容改造完成 + 本地验证。实现完成后，是否 build/合并/推送触发 Render 自动部署，按此前已讨论过的发布清单（工作树干净度确认 → 测试 → build → 合并 main → push）单独向用户确认后再执行，不在本次设计/实现计划里默认包含推送到生产这一步。
