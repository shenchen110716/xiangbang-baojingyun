# Codex 项目协作规则

本文件适用于 Codex 在“响帮帮无忧保”仓库内的所有开发工作。

## 开始任务前

1. 完整阅读 `docs/AI-DEVELOPMENT-PROTOCOL.md`。
2. 阅读 `docs/ai-handoffs/` 中所有状态为 `active`、`blocked`、`review` 或 `ready` 的交接文件。
3. 运行 `bash scripts/ai_coordination_check.sh`。
4. 检查 `git status --short --branch`、`git worktree list`，以及其他活动分支相对 `main` 的修改文件。
5. 功能开发必须使用独立分支和独立工作树；不得直接在 `main` 上实现功能。

## 协作边界

- 不修改另一个代理的工作树。
- 不覆盖、不清理、不提交来源不明的本地文件或用户未提交修改。
- 一个任务只允许一个主负责人；后开始的任务遇到重叠文件时必须避让或标记为阻塞。
- 用户、认证、RBAC、计费、佣金、看板、公共类型、公共路由和数据库迁移属于串行修改区域。
- 数据库迁移必须接在最新已合并迁移之后；不得从旧基线创建平行迁移头。
- Python/Alembic 是数据库迁移的唯一权威；Java 同步实体与 Mapper，但不得建立第二套迁移历史。

## 交接与完成

- 开始、范围变化、阻塞、待审、合并后，都要更新对应的 `docs/ai-handoffs/<task>.md`。
- 提交应小而完整，并在交接中记录提交号、迁移、测试、风险和后续动作。
- 未通过协议规定的测试门槛，不得建议合并或发布。
- 未经用户对该次发布明确授权，不得部署生产环境、修改生产密钥或上传/提交微信小程序版本。
