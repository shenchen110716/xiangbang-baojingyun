# 响帮帮无忧保｜新版产品设计文档

版本：v2.0 · 保险经纪运营管理平台

## 0. 端产品规划

系统分为电脑端和移动端：

| 端 | 使用人 | 主要职责 |
| --- | --- | --- |
| 电脑端·平台端 | 平台运营、财务、理赔、业务负责人 | 保司、产品、单位、岗位、员工、保单、参停保、佣金和大屏 |
| 电脑端·用户端 | 投保企业 / HR 机构 | 用工单位、岗位、员工、参保停保、保单、账户和理赔 |
| 移动端·用户端 | 企业管理员 / HR 管理员 | 快速新增、批量导入、参保停保审批、员工编辑、消息和分享 |

移动端使用微信小程序实现，登录后按 `enterprise_id` 隔离数据；分享使用微信小程序分享卡片，分享内容只包含安全的业务链接或邀请参数，不直接暴露员工证件和保单敏感信息。

## 1. 产品定义

响帮帮无忧保是一个连接保险公司、保险经纪平台、投保单位、业务员和参保员工的保险经纪业务系统。平台以保险公司方案为产品主数据，以投保单位为客户主体，以岗位和参保员工为业务执行对象，最终沉淀保单、保费、理赔和佣金数据。

核心业务链路：

```text
保险公司方案 → 投保单位 → 用工单位 / 岗位 → 参保员工 → 保单 → 理赔 / 结算
                         ↘ 业务员推广 / 佣金关联 ↗
```

## 2. 门户设计

### 2.1 总后台

面向平台管理员、运营、财务、理赔和业务负责人，管理全平台数据。

一级模块：

- 工作台 / 经营大屏
- 业务管理：投保单位管理、岗位管理、参保员工管理、劳动关系管理
- 保险产品管理
- 投保单位管理
- 参保员工管理
- 投保岗位管理
- 保险公司
- 保单管理
- 参停保中心
- 推广与佣金
- 报表中心
- 资金与发票
- 消息中心
- 账户设置

### 2.2 参保单位后台

面向企业或 HR 机构，只能访问自身租户数据：

- 用工单位
- 投保岗位
- 职业类别
- 参保员工
- 参保产品
- 保单
- 理赔
- 保费账户 / 使用费账户
- 单位管理员

用户端参保 / 停保必须同时支持两种入口：

- **手工添加**：填写员工姓名、手机号、证件信息、实际用工单位、投保岗位、职业类别和产品方案，提交人工审核。
- **批量导入**：下载标准 Excel 模板，上传后进行字段校验、重复校验、职业类别校验和错误行下载；校验通过后按批次提交。

员工列表必须提供：

- 员工信息编辑
- 参保
- 停保
- 恢复参保
- 批量勾选与批量参停保
- 参停保记录和批次详情
- 保单、产品和投保关系查看

### 2.3 移动端用户端

微信小程序页面：

- 首页：在保人数、待审核、余额和快捷操作
- 员工列表：搜索、状态筛选、分页 / 下拉加载
- 手工新增：员工与岗位表单
- 批量导入：模板下载、文件上传、校验结果、提交批次
- 员工详情：编辑、参保、停保、保单和操作记录
- 参停保批次：批次状态、错误记录、回执
- 参保产品：单位可用方案和价格
- 消息中心：审核结果、余额预警、保单到期
- 分享：分享单位投保邀请、岗位招募或安全的业务详情

小程序分享规则：

```text
分享卡片 → 安全短 token → 登录 / 授权 → 校验企业权限 → 展示业务页面
```

参保单位不能访问保险公司维护、平台产品录入、全平台佣金和其他单位数据。

## 3. 总后台核心模块

### 3.1 保险产品管理

产品来自保险公司方案库，字段包括：

- 保险公司
- 产品方案名称
- 保障范围与保障额度
- 适用职业类别
- 基础价格
- 返佣比例
- 支付模式：企业直投 / 平台大保单
- 在售 / 暂停状态

功能：

- 新增、编辑、暂停、恢复
- 产品搜索和记录统计
- 查看绑定的投保单位、参保人数、保费总额
- 已被保单或参保人员使用的方案不可删除，只能暂停

### 3.2 投保单位管理

单位类型包括最终用工企业和 HR / 派遣机构。

字段包括：

- 单位名称
- 单位类型
- 统一社会信用代码
- 联系人和联系电话
- 负责业务员
- 人工核验状态
- 保费核验账户
- 平台使用费账户
- 余额预警状态

功能：

- 新增、编辑、删除
- 负责业务员分配
- 单位管理员管理
- 保费账户充值
- 使用费账户充值
- 单位搜索
- 参保产品二级页面
- 参保人数和保费汇总

删除规则：已有参保员工或保单的单位不可删除，必须先停保并归档。

### 3.3 参保员工管理

员工必须关联：

```text
投保单位 → 实际用工单位 → 投保岗位 → 职业类别 → 产品方案 → 保单
```

功能：

- 员工新增和首次人工审核
- 员工搜索、状态筛选、分页、导出
- 查看员工详情和保单
- 编辑员工资料
- 停保 / 恢复
- 批量操作
- 理赔入口

### 3.4 投保岗位管理

岗位字段：

- 所属投保单位
- 实际用工单位
- 岗位名称
- 职业类别
- 绑定产品方案
- 保司核保状态
- 在保人数

功能：

- 新增岗位
- 修改岗位
- 删除岗位
- 绑定 / 更换产品方案
- 提交保司核保
- 按职业类别筛选

### 3.5 业务员管理与推广佣金

业务员基础资料与佣金规则分离。

业务员资料：

- 姓名
- 登录账号
- 初始密码
- 手机号码
- 启用 / 停用状态

业务佣金关系：

```text
业务员 + 投保单位 + 产品方案 + 佣金比例 + 业务状态
```

功能：

- 新增业务员
- 启用 / 停用业务员
- 新增业务关系
- 查看业务关系列表
- 业务员、单位、产品和佣金比例筛选
- 最高返佣比例统计
- 佣金金额和结算单

佣金模式：

- 企业直投：保险公司返佣中按规则给业务员提成，不允许业务员加价。
- 平台大保单：平台收款，业务员可加价销售，加价部分按规则计入业务员佣金。

## 6. 数据模型

- `User`：管理员、单位管理员、业务员
- `Enterprise`：投保单位、单位类型、账户、负责业务员
- `InsurancePlan`：保险公司产品方案
- `WorkPosition`：投保岗位和职业类别
- `InsuredPerson`：参保员工
- `Policy`：保单
- `Claim`：理赔
- `AgentCommission`：业务员 + 单位 + 产品方案 + 佣金比例
- `AuditLog`：所有关键操作审计

## 7. 权限原则

- 总后台可管理全平台数据。
- 参保单位只能查看自身 `enterprise_id` 数据。
- 业务员只能查看被分配的单位和业务关系。
- 产品删除、单位删除、停保、充值、佣金修改必须二次确认。
- 所有新增、编辑、删除、充值、关联和状态变更写入审计日志。

## 11. v3.0 全新总体架构（当前重构基线）

### 11.1 三端职责边界

| 端 | 服务对象 | 核心原则 | 一级功能 |
|---|---|---|---|
| 平台端 | 经纪公司运营、财务、核保、客服 | 全平台可见、规则配置、风险控制 | 工作台、经营大屏、产品与保司、投保单位、参保员工、岗位、保单、参停保、理赔、资金发票、报表、业务员、系统设置 |
| 投保单位电脑端 | 企业/HR 管理员 | 仅本单位数据，只能使用平台分配方案 | 工作台、用工单位、岗位、参保员工、参停保、保单、理赔、资金与发票、报表、单位设置 |
| 微信小程序端 | HR/授权经办人 | 移动高频操作、消息触达、分享 | 首页、员工、批量导入、参停保批次、平台产品、资金、理赔材料、消息 |

平台端负责“配置和审核”，投保单位端负责“业务执行”，小程序负责“移动采集和进度跟踪”。产品、价格、佣金、预警阈值等平台规则不在用户端维护。

### 11.2 领域模块拆分

1. **租户与权限域**：用户、角色、单位、业务员、数据范围、审计日志。
2. **产品与费率域**：保险公司、产品方案、职业类别子方案、按月/按天计费、次日/即时生效、推广和佣金。
3. **企业业务域**：投保单位、用工单位、岗位、职业类别、参保员工、劳动关系。
4. **保单与批次域**：保单、参保批次、停保批次、保司发送、回执和重试。
5. **资金结算域**：保费账户、平台使用费账户、充值订单、支付回调、对账、发票。
6. **理赔服务域**：报案、材料、补件、审核、赔付、拒赔、通知和操作轨迹。
7. **消息与预警域**：余额告罄 3–7 天预警、保单到期、批次回执、理赔状态、短信和邮件。

### 11.3 关键业务流程

```text
平台录入保司方案
  → 配置职业类别价格、计费模式、生效模式
  → 给投保单位关联方案和唯一业务员
  → 单位选择岗位并导入员工
  → 生成参保/停保批次
  → 保司 API 发送 → 回执/失败重试
  → 生成/更新保单 → 按实际天数核算保费
  → 保费账户与平台使用费账户分别扣费
  → 余额按当前规模预测 → 3–7 天预警
```

### 11.4 统一计费规则

- 按月方案：`月保费 × 当月实际保障天数 ÷ 当月自然日数`。
- 按天方案：`日费率 × 实际保障天数`。
- 次日生效：提交日不计费，次日 00:00 起算。
- 即时生效：审核通过/保司确认时间起算。
- 平台使用费：`在保人数 × 单位自定义日单价 × 实际在保天数`，默认 `0.1 元/人/天`。
- 保费与平台使用费必须使用不同账户、订单和对账科目。

### 11.5 外部集成可靠性

- 所有外部请求使用 `request_id`、幂等键、签名校验和超时重试。
- 保司、短信、邮件、支付均通过 Provider 适配层接入，不在业务代码中写供应商 SDK。
- 发送状态统一为：待发送、发送中、成功、失败、待重试、已确认。
- 支付回调必须验签、幂等处理，成功后才入账。
- 真实环境使用 `INTEGRATION_MODE=real`，开发环境使用 mock 适配器。

### 11.6 权限矩阵

| 能力 | 平台端 | 投保单位端 | 小程序 |
|---|---:|---:|---:|
| 维护产品/费率 | ✓ | — | — |
| 分配单位产品 | ✓ | 查看 | 查看 |
| 维护参保员工 | ✓ | ✓ | ✓ |
| 参停保发送 | ✓ | ✓（本单位） | ✓（授权） |
| 充值/发票 | ✓ | ✓（本单位） | ✓ |
| 修改预警阈值 | ✓ | 按平台授权 | — |
| 理赔材料 | ✓ | ✓ | ✓ |
| 支付回调/对账 | ✓ | — | — |

### 11.7 技术落地

- 前端：电脑端 SPA + 微信小程序，共用接口契约和权限语义。
- 后端：FastAPI，按领域路由拆分，SQLAlchemy 数据模型，JWT 鉴权。
- 数据：生产环境 PostgreSQL；本地演示 SQLite；所有表保留 `created_at/updated_at` 和审计记录。
- 异步：发送、短信、邮件、回执和对账进入任务队列，失败可重试。
- 可观测性：结构化日志、请求编号、接口耗时、失败率、余额预警和外部回执监控。
- 安全：密钥仅从环境变量读取，生产启用 HTTPS、回调验签、敏感字段脱敏和最小权限。

### 11.8 v3 交付验收标准

- 三端菜单按角色自动隐藏，接口层同时执行数据权限。
- 任一单位只能关联一个业务员，但可关联多个产品方案。
- 参保、停保、充值、回调、理赔材料均有真实状态记录和审计日志。
- 所有列表具备搜索、筛选、分页、统计、导出、空状态和失败重试。
- 大屏展示单位数、在保人数、产品保费、账户余额、预计消耗和 3–7 天预警。

## 13. 《保经云问题.docx》问题修正版：v3.1 业务模型重构

### 13.1 核心模型修正

原系统把“投保单位、实际用工单位、岗位、保险方案”混在参保员工表单中，导致员工可以自行选择职业类别，无法满足劳务派遣业务。新版调整为固定业务链：

```text
投保单位（劳务派遣公司）
  → 实际用工单位（合作企业）
  → 投保岗位
  → 岗位审核视频 / 职业定类
  → 保险方案与职业类别价格
  → 参保员工
```

参保员工新增时只选择“实际用工单位 + 岗位”，系统自动带出岗位已经审核通过的保险方案、职业类别、费率和生效规则，员工不能自行修改类别或方案。

### 13.2 实际用工单位模块

新增“实际用工单位管理”，字段包括：

- 所属投保单位（劳务派遣公司）
- 实际用工单位名称、统一社会信用代码
- 联系人、电话、地址
- 合作状态、合同编号、合同有效期
- 关联岗位数、在保员工数、累计保费

平台端可新增、审核、停用和归档；投保单位端可维护本单位合作企业；员工、岗位和保单全部以实际用工单位为查询维度。

### 13.3 岗位审核与视频定类

岗位必须完成以下流程后才能参保：

1. 新增岗位名称和实际用工单位。
2. 上传岗位工作视频、岗位说明书和现场照片。
3. 平台审核视频真实性和工作内容。
4. 核保人员确定职业类别：1–3 类、4 类、5 类、超 5 类。
5. 绑定保险方案子类别和价格。
6. 审核通过后岗位状态变为“可参保”。

岗位状态：草稿、待视频、待审核、补充材料、已定类、已通过、已停用。

### 13.4 参保员工模块修正

- 被保险人核心身份字段使用身份证号码，不再使用手机号作为主身份字段。
- 手机号保留为通知联系方式，不作为职业类别或费率依据。
- 所在企业固定为实际用工单位。
- 投保岗位固定为已审核岗位。
- 保险方案、职业类别、费率由岗位自动带出，不允许员工表单自行选择。
- 必须显示生效日期、预计终止日期、实际保障天数和当前累计费用。
- 增加“减员/停保”操作，支持单个停保和批量停保。
- 状态统一为待审核、待生效、在保、停保中、已停保、拒保。

员工搜索条件增加：投保单位、实际用工单位、岗位、身份证号、姓名、在保状态、生效日期。

### 13.5 参保导出修正

参保和停保导出必须包含：

- 投保单位
- 实际用工单位
- 岗位名称和职业类别
- 被保险人姓名、身份证号
- 保险方案和保司
- 生效日期、停保日期
- 保障天数
- 原始保费、按天折算保费
- 平台使用费、累计费用
- 当前状态和保司回执号

导出支持按单位、实际用工单位、岗位、时间范围、状态和保司筛选。

### 13.6 保单管理修正

保单分为：

- 整体保单：按投保单位、保司和方案导出。
- 员工个人保单：按单个或批量员工导出。
- 实际用工单位保单：按合作企业导出。
- 岗位保单：按岗位和职业类别导出。

导出前必须选择导出范围，支持 PDF、Excel，并包含保障期限、保障天数、保费、保司保单号和回执状态。禁止默认只能导出整体保单。

### 13.7 理赔管理修正

理赔页面升级为案件工作台，显示：

- 案件号、投保单位、实际用工单位、岗位、被保险人
- 出险时间、报案时间、事故类型、事故描述
- 医院、诊断、就诊费用、申请赔付金额
- 所需材料清单、已提交材料、缺失材料
- 保司处理进度、当前节点、下一处理人、SLA 截止时间
- 补件记录、审核意见、赔付金额、赔付日期、拒赔原因

处理节点：报案、材料收集、已提交保司、保司审核、补充材料、核赔通过、已赔付、拒赔、结案。

### 13.8 页面体验修正

- 增大正文和表格字号，统一使用 14–16px 基础字号。
- 用颜色区分状态：在保绿色、待审核橙色、停保灰色、风险红色、已赔付蓝色。
- 重要提醒、缺失材料、余额风险使用醒目卡片。
- 所有复杂页面增加步骤条、面包屑、固定筛选栏和批量操作栏。
- 大屏保留深色视觉，但业务表格和理赔工作台使用高对比浅色面板，减少长时间操作疲劳。

### 13.9 v3.1 验收标准

- 员工新增无法自行选择职业类别，只能选择已审核岗位。
- 没有实际用工单位或岗位审核视频，不能新增参保员工。
- 员工可单个或批量减员，停保后不再产生后续保费和平台使用费。
- 导出文件包含保障天数和累计费用。
- 保单可选择整体、实际用工单位、岗位或个人员工导出。
- 理赔案件具备材料清单、补件、进度、审核意见和赔付闭环。
- 投保单位、实际用工单位、岗位和员工的查询关系可追溯。

## 15. 工伤理赔三端设计

### 15.1 状态机

```text
已报案 → 材料收集中 → 已提交保司 → 保司审核中
                          ↑              ↓
                          └── 待补充材料 ─┘
                                         ↓
                              核赔通过 / 拒赔
                                  ↓         ↓
                                已赔付     已结案
                                  ↓
                                已结案
```

状态只能按规定方向推进，每次变化记录操作人、时间、意见和审计日志。

### 15.2 材料清单

- 被保险人身份证明
- 劳动关系证明
- 医疗诊断证明
- 病历或出院记录
- 医疗发票和费用清单
- 事故经过及证明
- 收款银行卡信息

材料支持图片、PDF、Word、Excel，单个文件不超过 20MB；提交保司前必须齐全。

### 15.3 平台端

- 查看全平台案件、风险、材料完整度和 SLA。
- 审核事故信息和材料，发起补件。
- 登记保司报案号、审核意见、核赔金额、拒赔原因和赔付时间。
- 推进保司审核、核赔通过、赔付和结案节点。

### 15.4 投保单位电脑端

- 选择本单位员工发起报案。
- 填写事故时间、地点、类型、医院、诊断、金额和事故经过。
- 上传材料、查看缺失项和处理时间线。
- 材料齐全后提交保司审核，查看核赔和赔付结果。

### 15.5 微信小程序端

- 移动报案和案件列表。
- 拍照上传身份证明、医疗材料、发票和事故证明。
- 查看材料完整度、补件要求、当前节点和完整时间线。
- 材料齐全后一键提交保司审核。

### 15.6 时限提醒

- 单位申请工伤认定按事故日期计算 30 日提醒。
- 系统显示截止日期、剩余天数和超期风险。
- 平台可配置案件负责人和内部 SLA，临期进入消息中心和大屏提醒。
